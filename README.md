# 🌱 Lucid Hubble — 极简主义数字花园

一个基于 **Astro** 构建的高颜值、高性能、极简主义风格的个人博客系统。采用纯静态生成 (SSG) 与 Zero-JS default 技术栈，追求极致的纸质杂志排版美学与舒缓的暗黑模式自适应体验。

---

## ✨ 核心特性

* **🎨 极简杂志美学**：采用精挑细选的 HSL 调和色彩系统（Warm White / Pitch Black），具备精细调校的呼吸感行间距排版，完美适配手机、平板及大屏设备。
* **⚡ 极致首屏性能 (Zero JS)**：Astro 将 Markdown 在打包期即编译为纯静态 HTML 发送给客户端，没有任何运行时庞杂 JS 依赖，页面秒开。
* **💻 VSCode 同款高亮 (Shiki)**：内置 Shiki 构建期语法高亮，高亮颜色变量与全局 CSS 样式设计系统完美结合，亮暗色自适应，无运行时闪烁。
* **✍️ 零配置免维护写作**：原生支持内容集合（Content Collections）。写新文章只需在特定目录下放入 `.md` 文件，列表自动识别并以淡入动画倒序排列呈现，无需手动维护任何索引 JSON。
* **🌓 零闪烁暗黑模式**：内嵌防止页面闪烁 (FOUC) 的持久化暗黑模式微脚本，刷新页面能瞬间记住用户的主题偏好。
* **🚀 终身免费一键部署**：由于是纯静态零构建项目，支持 Vercel、GitHub Pages 等托管平台一键自动检测并零成本免费上线。

## 🛠️ 本地开发运行指引

项目仅依赖原生 **Node.js** 环境：

### 1. 安装项目依赖
```bash
npm install
```

### 2. 启动本地开发服务
```bash
npm run dev
```
启动后，在浏览器访问 `http://localhost:8000/` 即可实时预览并支持页面热更新 (HMR)。

### 3. 本地静态打包与预览
```bash
# 生产环境静态打包，产物将生成至 dist/ 文件夹下
npm run build

# 在本地 8000 端口预览打包出来的静态产物
npm run preview
```

## 📝 园丁写稿极简指引

当您想为您的数字花园添砖加瓦，撰写新文章时，仅需两步：

1. **写稿**：在 `src/content/blog/` 目录下创建一个新的 `.md` 文件，例如 `src/content/blog/my-new-post.md`。
2. **定义 Frontmatter**：在文章的最顶端加上文章的 YAML 元数据标签：
   ```markdown
   ---
   title: "我的新文章标题"
   date: "2026-05-28"
   description: "这是一句关于新文章的精美摘要。"
   tags: ["生活", "思考"]
   ---

   这里是您开始撰写的 Markdown 正文内容...
   ```
   *主页会自动捕获该文件，按时间倒序将新文章推送到首页顶部，并完成所有代码块的高亮处理。*
