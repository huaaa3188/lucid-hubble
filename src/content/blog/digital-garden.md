在前端技术日新月异、框架层出不穷的今天，我们似乎已经习惯了使用 Next.js, Nuxt 或是 Webpack/Vite 这样复杂的编译链路来构建即使是一个最简单的个人网站。

然而，当我们跳出技术惯性，重新审视“个人博客”的本质时，我们会发现——**它其实只需要把文字优雅地呈现给读者即可。**

今天，我们来聊聊如何通过最质朴的现代静态编译技术，重拾纯粹的工程美学。

---

## 极简静态构建与零运行时的优雅

现代大型前端框架为我们带来了依赖管理与组件化，但同时也随之带来了沉重的运行时运行时依赖。一旦多年后你需要修改一处拼写，可能需要先处理一长串早已失效的 `npm install` 报错。

Astro 极简静态网页编译拥有以下难以比拟的工程优势：

- **万物皆持久**：Astro 将内容直接编译成最纯净的静态 HTML，只要浏览器标准还在，十几年后的浏览器依旧能瞬间打开，无需担心客户端框架失效。
- **零运行时 JS 负担**：客户端加载时不附带任何无关的 JS（Zero JS by default），整体页面在弱网环境下也能瞬时完成首屏渲染。
- **零安全漏洞**：静态 HTML 意味着没有后端数据库与庞杂的运行时漏洞，这里没有任何可以被攻击或注入的入口。

## 精巧的 HSL 设计系统

在这个博客中，我们完全摒弃了像 Tailwind 这样的第三方样式框架。取而代之的是，我们使用原生现代 CSS 的 **HSL (Hue, Saturation, Lightness)** 变量定义了一套支持平滑无缝过渡的暗黑/亮色主题。

例如，亮色模式下我们使用略带暖意的主背景，而暗黑模式下我们采用低饱和度的纯黑：

```css
:root {
  /* 亮色模式：温润纸张感 */
  --bg: 36, 15%, 97%;          /* HSL: 略带暖调的砂白色 */
  --fg: 0, 0%, 11%;            /* 柔和黑 */
  --border: 36, 10%, 88%;
}

[data-theme="dark"] {
  /* 暗黑模式：深邃纯净黑 */
  --bg: 0, 0%, 4%;             /* 极致深邃黑 */
  --fg: 0, 0%, 90%;            /* 舒适灰白 */
  --border: 0, 0%, 15%;
}
```

而在转换机制上，我们只需要这一行简单的原生 JS：

```javascript
document.documentElement.setAttribute('data-theme', newTheme);
```

配合 CSS 的 `transition` 规则，页面所有色彩便会像灯光调暗般，在 0.4 秒内优雅而平滑地在两套系统间交替：

```css
body {
  transition: background-color 0.4s ease, color 0.4s ease;
}
```

## 优雅的代码排版 (Code Typo)

作为一名工程师，代码是我们的画笔。在详情页中，我们完全摒弃了在客户端加载庞大 JS 库渲染代码的旧做法。取而代之的是，我们在 Astro 构建期调用了 VSCode 同款的 **Shiki** 高亮渲染引擎，并且将高亮主题直接托管给全局 CSS 变量。

我们在 `astro.config.mjs` 中仅需极简的配置即可开启它：

```javascript
// astro.config.mjs
export default defineConfig({
  markdown: {
    // 启用 Shiki，使用 css-variables 模式
    // 这将自动把代码拆解为带有 CSS 变量的高雅 HTML 标签，完全杜绝客户端渲染带来的首屏闪烁
    shikiConfig: {
      theme: 'css-variables'
    }
  }
});
```

如此一来，客户端**零 JavaScript 加载开销**，代码框的主色调、关键字、字符串和注释的着色都和我们的 HSL 暗黑模式色彩完美融合。
