不知道你有没有这种感觉：现在的网页变得越来越重了。

有时候我只是想点开看一篇不到两百字的文章，浏览器却要默默加载好几兆的 JavaScript 包，甚至因为各种花哨的动画效果卡得我的电脑风扇呼呼作响。很多时候，技术上的“进步”似乎并没有让阅读体验变得更好，反而塞满了无谓的累赘。

写这个小站的时候，我决定给自己做一次彻头彻尾的“断舍离”。

---

## 不用框架，不用 Tailwind，只用最原始的 CSS

现在几乎所有人写样式都会默认引入一大堆 CSS 库。我这次反其道而行之，一行第三方样式都没用，全部手写。

为了让亮暗模式切换看起来不那么生硬，我用原生的 HSL 变量做了一套配色：

```css
:root {
  /* 亮色模式：像一张暖白色的旧纸张 */
  --bg: 36, 15%, 97%;
  --fg: 0, 0%, 11%;
  --border: 36, 10%, 88%;
}

[data-theme="dark"] {
  /* 暗色模式：没有刺眼光芒的深邃黑 */
  --bg: 0, 0%, 4%;
  --fg: 0, 0%, 90%;
  --border: 0, 0%, 15%;
}
```

不需要安装任何臃肿的主题切换插件。客户端只需要一行原生 JS 去更改属性，配合 CSS 的 `transition`，整个网页的色调就会像无级调光灯一样，在 0.4 秒里丝滑地过渡过去。

这种简单的美感，比套用几万行成型的样式框架要让我爽得多。

---

## 让客户端彻底“断网”

我以前经常遇到这种尴尬：很久以前写的一些前端项目，过几年想改个错字，结果 `npm install` 直接报错一大堆，说某些依赖早已失效。

静态页面（SSG）最牛逼的地方就在于“万物皆持久”。Astro 在打包时把所有的 Markdown 随笔直接翻译成了纯粹的 HTML 静态字符串。

这意味着，只要十几年后互联网标准还没崩溃，任何浏览器依然能在一瞬间加载出我的文字。没有后端数据库，没有需要持续运行的容器，这里也永远不需要担心被 SQL 注入或者暴露出任何安全漏洞。

---

## 总结

写这个博客的过程，让我重新找回了刚学编程时那份最朴素的快乐。

少一点无意义的抽象，多一点对本质的专注。代码其实可以很简单。
